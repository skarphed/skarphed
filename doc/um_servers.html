Servers

A server in the skarphed universe is something that has an IP. Simple as that. A server can be a Desktop PC, a Rackserver, a Virtual Machine, or even your Coffeemaker given the fact that it supports TCP/IP. Skarphed's components are hosted on servers. skarphed-admin talks to it's servers via SSH, so each server that is to be used in your skarphed infrastructure must deliver SSH at TCP-port 22

Adding a server

In skarphed-admin you add a server by clicking the Button Add in the toolbar and then filling up the register server dialog. Following fields must be filled out:

  • IP:

    Enter the IP of your Server in this field. Please do not use DNS-Names here. The server really should represent the physical machine here. Skarphed handles DNS-Names in the skarphed-core

  • Name:

    Give your server a name here, so that you do not have to remember the purpose of each machine by memerizing it's IP. For example if you want to use the server as databaseserver, call it DB1.

  • Username:

    This field expects you to enter the SSH username, respectively the UNIX-user you want skarphed-admin to log in as. Currently, skarphed can only act as the servers root user. For now just enter root here

  • Password:

    Enter the SSH-password of the UNIX-user, you entered into the field Username. As mentioned in Username skarphed-admin can currently only act as root.

Security stuff

Some people may be confused by the fact, that skarphed-admin needs a rootuser on the systems, because allowing root to login from remote is bad practice. First of all, this is a temporary state. There will support for other users be implemented. Skarphed needs superuser powers on the server to install programs such as webservers and python-libraries. However even with a required rootuser-remote-account, it is possible to create a secure skarphed environment. We assume, that the PC that runs skarphed-admin is located in the same network as the servers that you want to control with it, as it is the case in many corporate networks. You can configure your server in a manner, that it only accepts SSH-requests from a certain IP-range in that network, for example with iptables.
